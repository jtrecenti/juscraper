API Conventions

Standard Parameter Names

All scrapers in juscraper follow a unified API convention for parameter names.

Search

Parameter Description
pesquisa The search query string (canonical name)

Deprecated aliases: query, termo — accepted with DeprecationWarning, will be removed in v1.0.

Date Filters

Parameter Description
data_julgamento_inicio Start date for judgment filtering
data_julgamento_fim End date for judgment filtering
data_publicacao_inicio Start date for publication filtering
data_publicacao_fim End date for publication filtering

Generic aliases: data_inicio / data_fim map to data_julgamento_inicio / data_julgamento_fim. Cannot be used together with the specific names — raises ValueError.

Deprecated aliases: data_julgamento_de / data_julgamento_ate, data_publicacao_de / data_publicacao_ate — accepted with DeprecationWarning.

Accepted input formats

For endpoints with a wired pydantic schema (eSAJ family, 1C-a tribunals, TJDFT, TJES — see Schemas pydantic in CONTRIBUTING.md for the current list), the four canonical date fields accept any of:

  • "DD/MM/AAAA" — Brazilian short form (e.g. "01/03/2024")
  • "DD-MM-AAAA" — Brazilian with dashes (e.g. "01-03-2024")
  • "AAAA-MM-DD" — ISO-8601 short form (e.g. "2024-03-01")
  • "AAAA/MM/DD" — ISO-8601 with slashes (e.g. "2024/03/01")
  • datetime.date
  • datetime.datetime (the time component is dropped)

The pipeline coerces every recognized input to whatever format the tribunal’s backend expects — eSAJ uses DD/MM/AAAA, REST/JSON backends like TJDFT use AAAA-MM-DD. The exact backend format is declared per schema via the BACKEND_DATE_FORMAT: ClassVar[str] attribute, so callers don’t need to know it. An empty string and None pass through unchanged (open-ended search). An unrecognized format surfaces as a single ValueError("...Formato esperado: ...") raised by validate_intervalo_datas.

Pagination

Parameter Type Description
paginas int \| list \| range \| None Pages to download (1-based)
  • None (default): downloads all available pages
  • int: shorthand — paginas=3 is equivalent to range(1, 4)
  • range: e.g., range(1, 4) downloads pages 1, 2, 3
  • list: e.g., [1, 3, 5] downloads specific pages

Parameter Support by Tribunal

Parameter TJSP (cjsg/cjpg) TJDFT TJPR TJRS
pesquisa Yes Yes Yes Yes
data_julgamento_inicio/fim Yes Warned Yes Yes
data_publicacao_inicio/fim No Warned Yes Yes
paginas (all types) Yes Yes Yes Yes
paginas=None (all pages) Yes Yes Yes Yes

Migration Guide

From query / termo to pesquisa

# Before
scraper.cjsg(query="direito civil")
scraper.cjsg_download(termo="direito civil")

# After
scraper.cjsg(pesquisa="direito civil")
scraper.cjsg_download(pesquisa="direito civil")

From _de/_ate to _inicio/_fim

# Before
scraper.cjsg(query="teste", data_julgamento_de="01/01/2023", data_julgamento_ate="31/12/2023")

# After
scraper.cjsg(pesquisa="teste", data_julgamento_inicio="01/01/2023", data_julgamento_fim="31/12/2023")

From data_inicio/fim to data_julgamento_inicio/fim (TJSP)

# Before
scraper.cjsg("teste", data_inicio="01/01/2023", data_fim="31/12/2023")

# After (canonical)
scraper.cjsg("teste", data_julgamento_inicio="01/01/2023", data_julgamento_fim="31/12/2023")

# Or (generic alias still works with DeprecationWarning)
scraper.cjsg("teste", data_inicio="01/01/2023", data_fim="31/12/2023")

Pagination default change

# Before: default was paginas=1 (only first page)
scraper.cjsg(pesquisa="teste")           # downloaded 1 page
scraper.cjsg(pesquisa="teste", paginas=3) # downloaded pages 1-3

# After: default is paginas=None (all pages)
scraper.cjsg(pesquisa="teste")            # downloads ALL pages
scraper.cjsg(pesquisa="teste", paginas=1) # downloads page 1 only
scraper.cjsg(pesquisa="teste", paginas=3) # downloads pages 1-3

Date format flexibility

Wired endpoints accept multiple input formats and coerce to the backend’s expected representation. The four examples below all produce the same request:

import datetime as dt
import juscraper as jus

tjsp = jus.scraper("tjsp")     # eSAJ backend (DD/MM/AAAA)
tjdft = jus.scraper("tjdft")   # REST JSON backend (AAAA-MM-DD)

# String, any of the four supported formats — coerced internally:
tjsp.cjsg("dano moral", paginas=1, data_julgamento_inicio="01/03/2024")
tjsp.cjsg("dano moral", paginas=1, data_julgamento_inicio="2024-03-01")
tjdft.cjsg("habeas",    paginas=1, data_julgamento_inicio="01/03/2024")
tjdft.cjsg("habeas",    paginas=1, data_julgamento_inicio="2024-03-01")

# Or pass a date / datetime directly:
tjsp.cjsg("dano moral", paginas=1, data_julgamento_inicio=dt.date(2024, 3, 1))

Auto-chunking (eSAJ family)

Several eSAJ endpoints (cjpg/cjsg in TJSP, plus cjsg in TJAC, TJAL, TJAM, TJCE, TJMS) cap each request at a one-year data_julgamento_* window. When the user asks for a longer interval, the scraper automatically splits the request into year-long windows, fetches each one, and concatenates the results — deduplicating by cd_acordao (cjsg) or id_processo (cjpg).

import juscraper as jus

tjsp = jus.scraper("tjsp")

# Six-year span: scraper sends 6 requests under the hood and returns
# a single deduplicated DataFrame.
df = tjsp.cjsg(
    pesquisa="dano moral",
    data_julgamento_inicio="01/01/2018",
    data_julgamento_fim="31/12/2024",
)

The behaviour is controlled by auto_chunk: bool = True (default on). To restore the strict pre-2026 behaviour (a ValueError is raised for windows longer than 366 days), set auto_chunk=False — useful for input validation paths that want to fail before submitting a long job:

tjsp.cjsg(
    pesquisa="dano moral",
    data_julgamento_inicio="01/01/2018",
    data_julgamento_fim="31/12/2024",
    auto_chunk=False,  # raises ValueError ("aceita no máximo 366 dias")
)

Per-window failures (network errors, captchas, 5xx) are captured and surfaced as a single UserWarning listing the failed windows; the returned DataFrame contains whatever was downloaded successfully (parcial + warning). Combining auto_chunk=True with paginas is rejected when the interval is long enough to actually trigger chunking: the multi-window semantics for paginas are ambiguous (each window? aggregated budget?) and the design prefers to fail explicitly. For short intervals where chunking is a no-op, paginas works as usual.

Normalization Utilities

The juscraper.utils.params module provides normalization functions used internally by all scrapers:

  • normalize_paginas(paginas) — normalizes int | list | range | None
  • normalize_pesquisa(pesquisa, **kwargs) — handles deprecated aliases
  • normalize_datas(**kwargs) — handles deprecated and generic date aliases
  • coerce_brazilian_date(value, backend_format) — coerces any of the four supported string formats (or date / datetime) into the backend’s expected format
  • apply_input_pipeline_search(schema_cls, method_name, *, ...) — runs the canonical input pipeline (alias resolution → date coercion → interval validation → pydantic schema) for cjsg/cjpg endpoints
  • iter_date_windows(data_inicio, data_fim, *, max_dias=366) — splits a date range into non-overlapping windows for auto-chunking
  • run_chunked_search(fetch_window, *, data_inicio, data_fim, dedup_key, ...) — drives a search across windows, concatenates and deduplicates
  • warn_unsupported(param_name, tribunal) — warns about unsupported parameters
